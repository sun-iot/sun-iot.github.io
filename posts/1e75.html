解决

在真实的环境中是拿不到这个被重新赋值后的 id 值的。那问题来了，我一定要拿到，那我可以等这个 run 执行完去拿，那我怎么知道它啥时候执行完？只要时间不长，还是可以等的吗。而且呢，这个 Runnable 特别讨厌。我们看这个Runnable,她其实是一个函数式接口。从JDK1.0就跟着了，这还是元老级别的。难怪这么难用。然后呢，我们知道，Thread 是这个Runnable 的一个实现类。所以我们写线程，一般会用Thread，或者是Runnable。

笔者在网上挣扎了好久，看到有一个接口和Runnable很相似（以前学Java的时候，有看这块，但是时间长，忘了），叫 Callable 。大家看这个源码:

可以看到，最后是把结果 set 了一下。这个结果也就是 call 的返回结果，这样的话，我们不就相当于是等线程执行完就拿到结果了吗。这个时间还是可以等的。

步骤如下:

  • 实现一个 Callable 的类，重写里面的 call 方法;
  • 将实现类的实例加入到 FutureTask 里面去;
  • 把 FutureTask 实例加入到线程 Thread里面去。
  • 获取值。使用 future.get() , 如果线程没执行完，这个将会被阻塞住。
